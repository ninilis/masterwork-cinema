## Используемый стек технологий

### Frontend
- **HTML5** — семантическая разметка страниц (`index.html`, `hall.html`, `payment.html`, `ticket.html`, `admin.html`, `login.html`).
- **CSS3** — стилизация интерфейса, включая кастомные свойства (CSS-переменные) для единообразного управления цветами, отступами, шрифтами, а также медиазапросы для адаптивности.
- **JavaScript (ES6+)** — вся клиентская логика: динамическая отрисовка данных, обработка событий, drag-and-drop, взаимодействие с API, работа с локальным хранилищем (`localStorage`), управление модальными окнами.
- **[Bootstrap 5](https://getbootstrap.com)** — использование готовых компонентов (сетка, модальные окна, кнопки, формы) для ускорения вёрстки и обеспечения адаптивности.
- **[SortableJS](https://sortablejs.github.io/Sortable/)** — библиотека для реализации drag-and-drop при создании сетки сеансов (перетаскивание фильмов на временные ленты залов).
- **[QRCreator](https://github.com/lachenmayer/qr-creator)** — генерация QR-кода на странице билета (`ticket.html`).

### Backend / API
- **REST API** — взаимодействие с сервером через предоставленный эндпоинт [`https://shfe-diplom.neto-server.ru/`](https://shfe-diplom.neto-server.ru/). Использовались методы `GET` и `POST` для:
    - получения данных (фильмы, залы, сеансы);
    - авторизации администратора;
    - добавления/удаления сущностей;
    - обновления схемы залов и цен;
    - покупки билетов.
- **FormData** — для отправки данных в формате `multipart/form-data` (например, при загрузке постера или передаче конфигурации зала).

### Инструменты разработки
- **WebStorm** — среда разработки.
- **Git / GitHub** — система контроля версий, хранение кода.
- **Браузерные DevTools** — отладка, тестирование адаптивности, анализ сетевых запросов.
- **Chrome / Yandex / Microsoft Edge** — для кросс-браузерной проверки.

## Заключение
Все технологии подобраны с учётом требований дипломного проекта: используется чистый JavaScript без фреймворков, внешние библиотеки задействованы только там, где это оправдано (drag-and-drop, QR-код). Реализовано полное взаимодействие с предоставленным API.
